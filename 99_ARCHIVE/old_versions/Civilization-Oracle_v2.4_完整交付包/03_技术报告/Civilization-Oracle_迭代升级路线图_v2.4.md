## 执行摘要

Civilization-Oracle 项目在概念创新（四诊合参计算化、中华文明语义深度分析）上具有独特学术价值，但当前统计基础、数据质量和系统架构均不满足同行评审标准。本路线图整合五个专项深度研究成果，将升级任务分为 P0-P3 四个优先级，预计 36 个月内完成从原型到可发表研究系统的蜕变。

**核心战略转向**：从"文明预测系统"重新定位为**"中华文明语义深度分析平台"**——在 Turchin/Cliodynamics 已建立 30 年先发优势的预测领域正面竞争不可行，但在其从未触及的**语义分析深度**上具有独特差异化空间。

## 三、P0 任务执行指南（前30天可立即开始的5件事）

### 第1周

1. **导出原始回归输出**：从 phase6_pipeline.py 提取所有回归系数、样本量、R² 值
2. **编写 Adjusted R² + Cohen's d 函数**：参照 Track 1 §1C 代码片段，存入 `utils/stats.py`
3. **申请 CHGIS v6 数据访问权限**（track 2 建议截止日期 2026-06-01）
4. **搭建 pytest 测试框架**：为核心 Agent 选择测试路径（TextAnalyst / KGraph / Predictor）
5. **启动 CPM-KB 标注员培训**：参照 Track 2 §2B 标注规范手册

### 第2周

1. **运行 WFA 验证**：使用北宋 7 个历史周期数据，验证 PSI 公式的时间泄露问题
2. **构建 CHGIS 地名映射表**（部分北宋府州，Track 2 提供 18 个初始映射）
3. **SikuBERT 模型下载 + 编码基准测试**（Track 3 预期词汇覆盖率 91.7%）
4. **为 phase5_kgraph.py 编写第一个单元测试**（图构建函数）
5. **整理所有硬编码参数清单**：YAML 配置化第一步

## 四、与 v2.3 的核心差异

| 维度 | v2.3 | v2.4 | 差异说明 |
|------|------|------|---------|
| 统计方法 | 频率派，k折CV | 贝叶斯层次模型 + WFA | 小样本稳健性大幅提升 |
| 学术定位 | "预测系统" | "语义分析平台" | 差异化战略转向 |
| 数据工程 | CBDB 直连 | CHGIS 集成 + IPW 加权 | 地理覆盖 + 偏差校正 |
| NLP组件 | PEN (α=0.24) | SikuBERT + FCCPSL (α≥0.65) | 古籍适配 + 情感可靠 |
| 知识图谱 | RE-GCN (MRR=29.6%) | CEGRL-TKGR (MRR≥35%) | 因果推断 + 偏差校正 |
| 架构依赖 | Neo4j + Redis | DuckDB + NetworkX + Celery | 零外部依赖，可用性99%+ |
| 话语体系 | 精确数字 | 概率区间 + 情景描述 | 伦理合规性 |
| 学术发表 | 无具体路径 | 阶梯式发表策略 | 4阶段目标期刊 |

## 五、风险缓解策略

| 风险 | 触发条件 | 缓解策略 |
|------|---------|---------|
| 降级螺旋加剧 | 技术债务 > 验证可信度 > 资源获取 | P0 优先清偿单元测试和配置化债务 |

| 伦理时间炸弹 | 公众传播脱离方法论语境 | 概率区间替代精确数字；IEEE EAD 审查 |
