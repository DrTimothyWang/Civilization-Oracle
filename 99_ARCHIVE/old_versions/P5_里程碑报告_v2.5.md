## 一、v2.5 核心变更：从"重资产"到"轻资产"

### 1.1 问题背景

v2.4 架构依赖本地模型（SikuBERT Transformer），存在三个问题：
1. **依赖缺失**： transformers 未安装，模型降级为 mock
2. **环境脆弱**：需要 CUDA/GPU，本地测试困难
3. **更新困难**：模型权重难以动态更新

### 1.2 v2.5 解决方案

```
本地模型(SikuBERT) → 云端API(MiniMax/通义/Claude)
单文件Pipeline → 多脚本协作
固定阈值 → 动态IPW校正
```

## 三、技术实现细节

### 3.1 API 客户端架构

```
APIClient (基类)
├── MiniMaxClient (api.minimax.chat)
├── TongyiClient (dashscope)
└── ClaudeClient (api.anthropic.com)

自动降级：API key 缺失 → mock 模式
```

### 3.2 PSI 计算公式（v2.5）

```
SFD = 0.5 * avg_sentiment_normalized + 0.5 * density_norm
GSI = 1.0 + (north_ratio - 0.5) * 0.8
PSI = SFD * GSI
PSI_IPW = PSI + IPW_bias
```

### 3.3 CBDB dynasty code 映射

```
CBDB c_dy = 6  → 唐朝 (生年 618-907)
CBDB c_dy = 15 → 宋 (生年细分: 北宋960-1027/1028-1127, 南宋1128-1279)
CBDB c_dy = 19 → 明朝 (生年 1368-1644)
```

### 3.4 数据质量标签

```
A: 完整（生卒年+姓名+籍贯） → score >= 7
B: 良好（有生年+姓名） → score >= 4
C: 一般（有生年或姓名之一） → score >= 2
D: 缺失较多 → score < 2
```

## 四、使用方法

### 4.1 数据导出（cbdb_import.py）

```bash
python cbdb_import.py                          # 导出全部5个朝代
python cbdb_import.py --dynasty 北宋前期       # 单独导出
python cbdb_import.py --check                  # 仅检查连接
```

### 4.2 PSI 分析（psi_pipeline.py）

```bash
python psi_pipeline.py                          # 全量分析5个朝代
python psi_pipeline.py --dynasty 明朝 --limit 100  # 单朝代限流测试
python psi_pipeline.py --dry-run               # 仅验证数据流
python psi_pipeline.py --api minimax           # 指定API供应商
```

### 4.3 论文生成（paper_assist.py）

```bash
python paper_assist.py                          # 生成完整论文v0.4
python paper_assist.py --chapter methods        # 仅方法章节
python paper_assist.py --psi-results output/psi_all_summary.json
```

### 4.4 Pipeline 运行（phase_pipeline_v2.py）

```bash
python phase_pipeline_v2.py                    # 单周期模式
python phase_pipeline_v2.py --multi            # 全周期扩展分析
python phase_pipeline_v2.py --api auto          # 使用auto API
```
