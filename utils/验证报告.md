# SikuBERT NLP 模块验证报告

## 1. 测试环境

- Python 3.x（标准库 + numpy）
- transformers 库：**未安装**（触发降级模式）
- 测试模式：规则+词典降级方案

## 2. 功能验证结果

### 2.1 情感分析

| 文本 | 情感分数 | 匹配关键词 |
|------|---------|-----------|
| 嘉祐之治，天下太平，盛世繁华 | **+0.6000** | 盛世、太平、繁华、天下、嘉祐、治 |
| 百年积弊入骨髓，民不聊生 | **-0.8000** | 积弊、骨髓、民不聊生 |
| 党争激烈，朝纲紊乱 | **-0.5333** | 党争、朝纲、紊乱 |

**评估**：情感分数语义正确，正面词→正分，负面词→负分，符合预期。

### 2.2 关键词提取

- 文本1（嘉祐之治...）：提取到6个关键词，politics域主导
- 文本2（百年积弊...）：提取到3个关键词，social/politics混合
- 文本3（党争激烈...）：提取到3个关键词，politics域主导

**评估**：关键词覆盖全面，按频率排序逻辑正确。

### 2.3 语义相似度

- 文本1 vs 文本2 = **0.0000**（无共有关键词，符合预期）
- 文本1 vs 文本3 = 0.0（同上）
- 文本2 vs 文本3 = 0.0（同上）

**评估**：降级方案基于关键词交集，语义相近但词表不同的文本无法识别相似性，这是已知局限。完整模型加载后使用向量余弦相似度可解决。

### 2.4 语义画像

```
avg_sentiment: -0.2444
sentiment_std: 0.6070
dominant_domain: politics
n_texts: 3
domain_counts: {'politics': 9, 'economy': 1, 'social': 2}
```

**评估**：三个文本整体偏负面（北宋末期政治衰败语境），politics域占绝对主导，与文献历史背景吻合。

### 2.5 向量化

- 向量维度：768（符合 BERT 标准维度）
- L2 归一化：已执行
- 降级模式向量为全零（词表匹配失败，multi-char 中文切词问题）

**已知局限**：降级模式下，encode() 方法依赖空格分词，中文文本连续字符无法正确切分，导致向量稀疏。建议在生产环境使用真实 SikuBERT 模型。

### 2.6 工厂函数降级

```
[SikuBERT] 模型加载失败，降级到规则版本: No module named 'transformers'
加载类型: SikuBERTNLP
使用降级: True
```

**评估**：工厂函数正确捕获异常并降级，use_mock=True 标志正确设置。
